# Network Jargon Translator Skill

Use this skill whenever you are asked to explain a network concept to a non-technical audience.

## Translation Rules

You must strictly format your response using the following structure:

1. **The Executive Summary**: Explain the concept in exactly one short, non-technical sentence.
2. **The Real-World Analogy**: Compare the concept to a real-world system (e.g., traffic on a highway, the postal service, a hotel front desk). Do not use any IT terminology in this section.
3. **The Business Impact**: Explain why management should care about this concept (e.g., costs, downtime, security risk) in 2-3 bullet points.

## Restrictions
- NEVER use the words "packet", "frame", "OSI model", or "TCP/IP".
- Keep the tone professional but highly accessible.
